ABOUT

This application is a demonstration of an attempt to create an intuitive geolocation-based mobile multiplayer game framework using HTML5.

The client can create a new game or connect to an already existing one and send frequent updates to the server over a websocket connection. The server will respond with the current game state, allowing the client to draw the new information to the canvas map. Game logic is handled on the server.

Implementing a new game type is fairly straightforward. Simply implementing a new class that extends the Game class and adding it to the list of game types on the front page will allow players to begin playing. All of the networking is ready to go.
